# Contributing

Pull requests are welcome.

## Expectations

- Keep changes aligned with the project's scope and goals.
- Make changes reviewable and reasonably easy to understand.
- Prefer small, focused pull requests over broad unrelated rewrites.
- Explain behavioral changes clearly in the pull request description.
- Security-sensitive changes should include careful reasoning and testing notes.

## Submission rules

- Do not include third-party binaries unless they are clearly required and
  their origin and license are documented.
- Do not add obfuscated files, generated blobs, or opaque dependencies that are
  difficult to audit.
- Do not submit changes that relicense third-party code under the Phantom
  Mirror project license.

## Contribution license and rights

By submitting a pull request, you confirm that:

- you have the right to contribute the submitted material
- the contribution is your own work or you otherwise have sufficient rights to
  submit it
- you allow the maintainer to use, modify, adapt, distribute, and publish the
  contribution as part of Phantom Mirror under the project license and future
  project license versions

Contributions do not change the ownership of the repository, the branding
rules, or the official-release rules.

## Official releases

Official Phantom Mirror releases are created only by the maintainer. A merged
pull request does not authorize a contributor or third party to publish an
official Phantom Mirror build.
